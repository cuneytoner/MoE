# Reference Board Markdown Download Implementation

## What Was Implemented

M34.27 adds a safe response-only Markdown download endpoint for reference boards.

The endpoint reuses the existing Markdown export helper and returns the Markdown as a browser attachment. It does not create runtime export files, ZIP files, PDF files, copied source assets, or generated media.

M34.28 mirrors the response-only attachment pattern for JSON.

M34.29 exposes Markdown download through dashboard UI.

M34.30 adds regression coverage for Markdown download.

## Endpoint

```text
GET /gateway/media/reference-boards/{board_id}/download/markdown
```

## Content-Type

```text
text/markdown; charset=utf-8
```

## Content-Disposition

The response includes:

```text
Content-Disposition: attachment; filename="reference-board-{board_id}-{YYYYMMDD-HHMMSS}.md"
```

## Filename Rules

The filename is generated by Gateway from:

- sanitized `board_id`
- UTC timestamp
- fixed `.md` extension

The filename does not use:

- board title
- user-supplied extension
- slash
- spaces
- dot-dot
- absolute paths
- shell-sensitive chars

Example:

```text
reference-board-api-test-board-20260709-150501.md
```

## Response-Only Behavior

The endpoint returns the attachment directly in the HTTP response.

It does not create:

- runtime export files
- `/home/cuneyt/MoE/runtime/reference-boards/exports`
- ZIP files
- PDF files
- copied source assets

## Safety Rules

- no arbitrary filesystem browsing
- no absolute host paths in exported content
- no source asset copy
- no source asset move
- no source asset delete
- no source asset rename
- no generation trigger
- no shell execution
- no ZIP
- no PDF
- no embedded images
- no embedded SVG
- no model files
- no secrets

## What Is Not Implemented Yet

- no JSON download endpoint
- no dashboard download button
- no ZIP export
- no PDF export
- no runtime export archive
- no controlled asset copy mode
