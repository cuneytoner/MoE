# Reference Board JSON Download Implementation

## What Was Implemented

M34.28 adds a safe response-only JSON download endpoint for reference boards.

The endpoint reuses the existing JSON export helper and returns the review pack as a browser attachment. It does not create runtime export files, ZIP files, PDF files, copied source assets, or generated media.

## Endpoint

```text
GET /gateway/media/reference-boards/{board_id}/download/json
```

## Content-Type

```text
application/json
```

## Content-Disposition

The response includes:

```text
Content-Disposition: attachment; filename="reference-board-{board_id}-{YYYYMMDD-HHMMSS}.json"
```

## Filename Rules

The filename is generated by Gateway from:

- sanitized `board_id`
- UTC timestamp
- fixed `.json` extension

The filename does not use:

- board title
- user-supplied extension
- slash
- spaces
- dot-dot
- absolute paths
- shell-sensitive chars

Example:

```text
reference-board-api-test-board-20260709-150501.json
```

## Response-Only Behavior

The endpoint returns the attachment directly in the HTTP response.

It does not create:

- runtime export files
- `/home/cuneyt/MoE/runtime/reference-boards/exports`
- ZIP files
- PDF files
- copied source assets

## Safety Rules

- no arbitrary filesystem browsing
- no absolute host paths in exported content
- no source asset copy
- no source asset move
- no source asset delete
- no source asset rename
- no generation trigger
- no shell execution
- no ZIP
- no PDF
- no embedded images
- no embedded SVG
- no model files
- no secrets

## What Is Not Implemented Yet

- no dashboard download button
- no ZIP export
- no PDF export
- no runtime export archive
- no controlled asset copy mode

## How To Test

### Run on PC-1
```bash
cd ~/DiskD/Projects/MoE/codebase
docker compose -f infra/docker/docker-compose.yml up -d --build gateway-api
```

### Run on PC-1
```bash
curl -fsS -D /tmp/moe-reference-board-json-download-headers \
  http://127.0.0.1:8100/gateway/media/reference-boards/api-test-board/download/json \
  -o /tmp/moe-reference-board-download.json
```

### Run on PC-1
```bash
grep -Ei 'content-type:|content-disposition:' /tmp/moe-reference-board-json-download-headers
jq '.export_type, .board.board_id, .board.item_count, .safety' /tmp/moe-reference-board-download.json
```

Expected:

- `Content-Type` includes `application/json`.
- `Content-Disposition` includes `attachment`.
- Filename starts with `reference-board-api-test-board-`.
- Downloaded JSON is valid.
- `export_type` is `reference_board_review_pack`.
- Safety flags show no asset copy/delete or generation.
- No runtime export file is created.
